Backend Development Specs

This section contains all the technical specifications, API contracts, and database schemas you need to implement the backend for this living platform.

API Endpoints

All endpoints should return JSON responses with proper HTTP status codes.

POST
/api/interactions
Records a user interaction with the platform
Request Body:
{
  "type": "click|hover|scroll|keypress|move",
  "coordinates": {"x": number, "y": number},
  "sessionId": "string",
  "userId": "string (optional)",
  "timestamp": "ISO string"
}
Response:
{
  "success": true,
  "interactionId": "string",
  "growthLevel": number (0-100)
}
GET
/api/platform/state
Returns current platform state and growth metrics
Response:
{
  "version": "1.0.3",
  "totalInteractions": number,
  "growthLevel": number (0-100),
  "lastUpdated": "ISO string",
  "activeExperiments": ["array of experiment IDs"]
}
GET
/api/platform/changelog
Returns recent platform evolution history
Response:
{
  "changelog": [
    {
      "id": "string",
      "message": "string",
      "timestamp": "ISO string",
      "triggeredBy": "system|user"
    }
  ]
}
POST
/api/contributions
Submit user contributions (experiments, art, code)
Request Body:
{
  "type": "experiment|art|code|idea",
  "title": "string",
  "content": "string",
  "userId": "string",
  "metadata": "object"
}
Database Schema

Recommended: PostgreSQL with the following tables:

platform_state

Column Type Description
idSERIAL PRIMARY KEYUnique identifier
versionVARCHAR(20)Current platform version
total_interactionsBIGINTTotal interactions across all users
growth_levelINTEGERCurrent growth level (0-100)
last_updatedTIMESTAMPWhen state was last updated

user_interactions

Column Type Description
idUUID PRIMARY KEYUnique interaction ID
user_idUUIDForeign key to users (nullable)
session_idVARCHAR(100)Anonymous session identifier
interaction_typeVARCHAR(20)Type of interaction
x_coordinateINTEGERMouse X position
y_coordinateINTEGERMouse Y position
created_atTIMESTAMPWhen interaction occurred

users

Column Type Description
idUUID PRIMARY KEYUnique user ID
emailVARCHAR(255)User email (unique)
display_nameVARCHAR(100)Public display name
total_contributionsINTEGERNumber of contributions
created_atTIMESTAMPAccount creation date
Implementation Notes
  • Session Management: Use localStorage to store anonymous session_id. When user logs in, associate session interactions with user account.
  • Rate Limiting: Implement rate limiting on /api/interactions to prevent abuse (max 100 interactions per minute per session).
  • Caching: Cache /api/platform/state responses for 5 seconds to reduce database load.
  • WebSocket Integration: For real-time updates, implement WebSocket endpoint that broadcasts growth level changes to all connected clients.
  • Error Handling: Return proper HTTP status codes (400 for bad requests, 429 for rate limiting, 500 for server errors).
  • Security: Validate all input, use parameterized queries to prevent SQL injection, implement CORS properly.
